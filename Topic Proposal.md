
# Topic Proposal

## 项目题目

上市银行ESG报告“绩效可核查性”评估：基于编制基础与披露透明度的跨年度分析（2022–2023）

## 金融问题

ESG数据可信性是投资者与监管机构的核心关切。本研究聚焦于上市银行ESG/可持续发展报告中对关键绩效指标（KPI）的**可核查性**——即报告是否提供了足够的“编制基础”信息（统计口径、范围、计算方法、鉴证情况），使得外部使用者能够评估数据的可靠性与可比性。具体回答：

1. 银行在绿色信贷、普惠金融、消费者权益保护等核心议题上，其绩效披露的**透明度**（是否有明确的编制基础、统计范围、计算方法）处于什么水平？
    
2. 从2022年到2023年，银行的披露透明度是**提升、停滞还是下降**？
    
3. 不同类型银行（国有大行、股份行、城商行）在可核查性上是否存在系统性差异？
    

## 巨潮数据来源

巨潮资讯网（[www.cninfo.com.cn](https://yuanbao.tencent.com/chat/naQivTmsDa/www.cninfo.com.cn)）

## 公告类型

- **ESG报告 / 可持续发展报告 / 社会责任报告**（属于临时公告或定期报告下的专项报告）

**数据规模**：计划覆盖 **80家A股上市银行**（含国有大行6家、股份行12家、城商行40家、农商行22家），每家银行收集 **2022年和2023年**​ 两个年度的ESG/可持续发展报告，合计 **约160份PDF**。实际样本以巨潮可下载到完整报告为准，预计不低于140份。

## 目标字段

记录粒度：**公司-年度-议题-锚点类型**。每个议题需在报告内定位三个锚点：**A.实质性矩阵**、**B.正文行动披露**、**C.绩效表及编制基础**。

| 字段组        | 字段名                   | 类型  | 必填  | 含义与抽取规则                                                 | 证据来源                 |
| ---------- | --------------------- | --- | --- | ------------------------------------------------------- | -------------------- |
| **基础**​    | `company_code`        | 文本  | 是   | 股票代码                                                    | 报告封面                 |

|            | `anchor_type`         | 分类  | 是   | `matrix`/ `narrative`/ `kpi`                            | 由section routing定位   |
|            | `source_page`         | 整数  | 是   | 证据所在PDF页码                                               | 解析时记录                |
|            | `evidence_snippet`    | 文本  | 是   | 支撑判断的原文片段（1-3句）                                         | 从定位区域截取              |
| **A.矩阵**​  | `in_material_matrix`  | 布尔  | 是   | 该议题是否出现在“实质性议题分析/矩阵”章节中                                 | 矩阵章节关键词匹配            |

## 最终输出形式

1. **结构化数据库**：
    
    - `base_records.csv`：约960条记录（80家×2年×6议题），含所有字段及证据页码。
        
    - `issue_trend.csv`：聚合到公司-议题粒度的趋势表，含两年分数和趋势标签。
        
    
2. **分析报告与可视化**：
    
    - **银行-议题可核查性热力图**（分2022、2023年两张）。
        
    - **“言行偏离”议题分布图**：高重要性但低可核查性的议题占比。
        
    - **银行披露透明度排名与年度变化图**。
        
    - **抽样评估报告**：对10%的记录进行人工校验，报告章节定位准确率、字段判断准确率。
        
    
3. **可复现的工程代码**：
    
    - 包含PDF解析、section routing（基于目录+关键词）、字段抽取（规则为主+LLM辅助）、跨年匹配、评分计算、评估脚本。
        
    - 所有代码遵循受控Vibe Coding六步法，并有完整的`ai_worklog.md`记录。
        
    

## 难度档位

- **1.1（挑战档）**
    
- **理由**：
    
    1. **数据规模**：处理约160份PDF，远超标准档下限，接近挑战档上限。
        
    2. **事件复杂度**：每个银行-年度构成一个“复杂分析事件”，内部需完成6个议题×3个锚点的定位、抽取与判断，实际分析单元超过960个。且需跨年匹配（同一银行不同年份的同一议题），形成纵向趋势分析。
        
    3. **工程链完整**：项目包含**PDF解析→章节路由→多锚点定位→规则+LLM混合抽取→跨年关联→评分计算→人工抽样评估**，技术栈覆盖课程所有核心要求。
        
    4. **评估与可追溯**：每个字段均绑定证据页码和原文片段，并计划对10%样本进行人工校验，确保结果可验证。
        
    

## 风险与备选方案

1. **风险**：部分银行（尤其是中小银行）可能未发布独立ESG报告，或报告年份不全。
    
    - **备选**：优先保证国有大行和股份行的完整覆盖；对于城商行/农商行，若某年缺失则仅保留有数据的年份，并在metadata中注明。最终样本量不低于140份PDF即可。
        
    
2. **风险**：PDF解析后表格结构丢失，导致绩效表数据难以抽取。
    
    - **备选**：对绩效表区域采用“关键词+邻近文本”的宽松策略，只抽取`has_kpi_value`布尔字段和文本形式的`kpi_value`，不强制精确数值。必要时辅以人工补录。
        
    
3. **风险**：LLM对`risk_tone`分类可能存在偏差。
    
    - **备选**：设计明确的few-shot示例，并对LLM输出进行人工抽样校验，将准确率作为项目报告的一部分。若准确率低于70%，则降级为仅使用规则字段。
        
    
4. **风险**：160份PDF的下载与解析耗时较长。
    
    - **备选**：分两批进行：第一批40家银行（80份PDF）作为核心样本，完成全流程分析；第二批40家银行作为扩展样本，仅完成关键字段抽取，用于验证结论稳定性。
